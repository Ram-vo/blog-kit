# Security Policy

If you find a security vulnerability, do not disclose it in a public issue.

Until a dedicated reporting channel is defined, treat all security findings as private.

## Initial Scope

The most sensitive areas will be:

- authentication adapters
- external provider integrations
- HTML/MDX content handling
- editorial permissions and RBAC

Before the first public release, the project should define:

- a security contact email
- a response SLA
- a responsible disclosure policy
